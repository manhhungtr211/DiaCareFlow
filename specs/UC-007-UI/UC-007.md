# UC-007: Giao diện Chat ReactJS

**Feature ID**: `UC-007`

**Input**: BR-002 (MVP Tuần 2)

**Phụ thuộc**: UC-006 (FastAPI Backend REST API)

---

## Actor

- **Người dùng cuối**: Mở trình duyệt, gõ câu hỏi về tiểu đường và xem câu trả lời.
- **Developer**: Sử dụng giao diện để test luồng E2E (UI: Reactjs đơn giản -> API -> Agents).

## Trigger

Người dùng mở giao diện web trên trình duyệt, gõ câu hỏi vào ô input và nhấn Enter hoặc nút gửi.

## Preconditions

1. Backend FastAPI (UC-006) đang chạy và endpoint `POST /api/chat` hoạt động.
2. CORS đã được cấu hình trên Backend cho phép gọi từ giao diện.
3. Trình duyệt Chrome hoặc Firefox phiên bản hiện tại.

## Main Flow

1. Người dùng mở giao diện web trên trình duyệt (localhost).
2. Người dùng gõ câu hỏi vào ô input (ví dụ: "Tiền tiểu đường là gì?").
3. Người dùng nhấn Enter hoặc nút gửi.
4. Giao diện hiển thị câu hỏi của người dùng trong vùng chat.
5. Giao diện hiển thị trạng thái loading (spinner/text).
6. Giao diện gửi `POST /api/chat` đến Backend với `{"question": "..."}`.
7. Backend trả về câu trả lời.
8. Giao diện hiển thị câu trả lời trong vùng chat, bên dưới câu hỏi.

## Alternative Flows

- **3a.** Người dùng nhấn Enter với ô input rỗng → Không gửi request, hiển thị gợi ý nhập câu hỏi.
- **7a.** Backend trả `is_refused: true` → Giao diện hiển thị câu từ chối với styling khác biệt (viền đỏ hoặc icon cảnh báo).
- **8a.** Người dùng gửi thêm câu hỏi → Cặp hỏi-đáp mới hiển thị bên dưới cặp trước đó, câu mới nhất ở dưới cùng.

## Exceptions

- **E1**: Backend không chạy hoặc không kết nối được → Giao diện hiển thị thông báo lỗi thân thiện: "Không thể kết nối đến server. Vui lòng thử lại sau."
- **E2**: Backend trả HTTP 500 (lỗi nội bộ) → Giao diện hiển thị: "Đã xảy ra lỗi hệ thống. Vui lòng thử lại."
- **E3**: Câu trả lời rất dài → Vùng hiển thị có scroll, không bị tràn layout.
- **E4**: Mạng chậm, response mất lâu → Loading indicator hiển thị rõ ràng, nút gửi bị disable để tránh gửi trùng.

## Acceptance Criteria

1. **Given** giao diện web đã mở và Backend đang chạy,
   **When** người dùng gõ "Tiền tiểu đường là gì?" và nhấn Enter,
   **Then** hiển thị loading, sau đó hiển thị câu trả lời từ Backend trong vùng chat.

2. **Given** giao diện web đã mở và Backend đang chạy,
   **When** người dùng gõ "Kê đơn thuốc cho tôi",
   **Then** hiển thị câu từ chối với styling khác biệt (viền đỏ/icon cảnh báo).

3. **Given** giao diện web đã mở nhưng Backend KHÔNG chạy,
   **When** người dùng gửi câu hỏi,
   **Then** hiển thị thông báo lỗi kết nối thân thiện (không hiện blank screen).

4. **Given** giao diện web đã mở,
   **When** người dùng nhấn Enter với ô input rỗng,
   **Then** không gửi request, hiển thị gợi ý nhập câu hỏi.

5. **Given** giao diện web đã mở,
   **When** người dùng nhìn vào trang,
   **Then** thấy tên "DiaCareFlow" ở header, bảng màu xanh dương/xanh lá/trắng (branding y tế).
