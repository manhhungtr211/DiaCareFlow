# UC-006: FastAPI Backend REST API

**Feature ID**: `UC-006`

**Input**: BR-002 (MVP Tuần 2)

**Phụ thuộc**: UC-005 (Multi-Agent Pipeline LangGraph)

---

## Actor

- **Client (ReactJS UI)**: Gửi HTTP request để hỏi đáp và nhận câu trả lời.
- **Developer**: Sử dụng Swagger UI để test API, kiểm tra health check.

## Trigger

Client gửi HTTP request `POST /api/chat` chứa câu hỏi dạng JSON đến Backend.

## Preconditions

1. Pipeline Multi-Agent (UC-005) đã hoàn thành ít nhất Phase 1 (2 nodes) và expose hàm callable từ Python.
2. Qdrant server đang chạy và kết nối được.
3. LLM API key đã được cấu hình.

## Main Flow

1. Client gửi `POST /api/chat` với body `{"question": "..."}`.
2. Backend validate input đầu vào (question không rỗng, không quá 2000 ký tự).
3. Backend gọi pipeline Multi-Agent LangGraph với câu hỏi.
4. Pipeline trả về kết quả (câu trả lời hoặc câu từ chối).
5. Backend đóng gói kết quả thành JSON response gồm: text, is_refused, refuse_reason (nếu có), sources, metadata (thời gian xử lý).
6. Backend trả HTTP 200 với JSON response cho client.

## Alternative Flows

- **2a.** Input thiếu trường `question` hoặc body rỗng → Backend trả HTTP 422 (Validation Error) với thông báo lỗi rõ ràng.
- **2b.** Câu hỏi dài hơn 2000 ký tự → Backend trả HTTP 422 với thông báo giới hạn.
- **4a.** Pipeline trả kết quả `is_refused = true` → Backend vẫn trả HTTP 200 nhưng response chứa `is_refused: true` và `refuse_reason`.
- **6a.** Client gọi `GET /api/health` → Backend trả trạng thái hệ thống (server status, Qdrant connection).
- **6b.** Client truy cập `/docs` → Backend hiển thị Swagger UI tự động sinh bởi FastAPI.

## Exceptions

- **E1**: Pipeline LangGraph gặp lỗi nội bộ (LLM timeout, Qdrant disconnect) → Backend trả HTTP 500 với thông báo lỗi thân thiện, không leak stack trace.
- **E2**: Client gửi request quá nhanh (spam) → Backend xử lý tuần tự, không crash (chưa cần rate limiting chính thức).
- **E3**: CORS violation — ReactJS UI chạy trên port khác → Backend cấu hình CORS cho phép cross-origin requests.
- **E4**: Qdrant không khả dụng khi health check → Trả `{"status": "degraded", "qdrant": "disconnected"}`, HTTP 200.

## Acceptance Criteria

1. **Given** server FastAPI đang chạy,
   **When** client gửi `POST /api/chat` với `{"question": "Tiền tiểu đường là gì?"}`,
   **Then** server trả HTTP 200 với JSON chứa: câu trả lời, nguồn tài liệu, metadata.

2. **Given** server FastAPI đang chạy,
   **When** client gửi `POST /api/chat` với `{"question": "Kê đơn Metformin cho tôi"}`,
   **Then** server trả HTTP 200 với JSON chứa `is_refused: true` và `refuse_reason`.

3. **Given** server FastAPI đang chạy,
   **When** client gửi `POST /api/chat` với body rỗng,
   **Then** server trả HTTP 422 với thông báo validation error.

4. **Given** server FastAPI đang chạy,
   **When** client gửi `GET /api/health`,
   **Then** server trả HTTP 200 với trạng thái hệ thống.

5. **Given** server FastAPI đang chạy,
   **When** giao diện ReactJS (port khác) gọi `POST /api/chat`,
   **Then** request thành công (CORS không bị block).

6. **Given** server FastAPI đang chạy,
   **When** developer truy cập `/docs`,
   **Then** Swagger UI hiển thị đầy đủ endpoint, request/response schema.
